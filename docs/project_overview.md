## 3. Key Patterns & Concepts

### 3.1 Constant-arrival-rate load model

Mọi script k6 chính dùng executor `constant-arrival-rate`: duy trì **RPS cố định** trong `DURATION`, scale VU qua `PRE_VUS` / `MAX_VUS`.

```javascript
// File: k6/k6-rkcrm-cardinfo.js (line 32-46)
export const options = {
  scenarios: {
    rkcrm_cardinfo_load: {
      executor: "constant-arrival-rate",
      rate: RPS,
      timeUnit: "1s",
      duration: DURATION,
      preAllocatedVUs: PRE_VUS,
      maxVUs: MAX_VUS,
    },
  },
  thresholds: {
    http_req_failed: ["rate<0.05"],
    http_req_duration: ["p(95)<3000", "p(99)<8000"],
  },
};
```

**Ghi chú:** Threshold mặc định — fail rate < 5%, p95 < 3s, p99 < 8s (rkCRM/NewApp). Legacy `k6-api-post-xml.js` dùng p95 < 2s.

### 3.2 Random data từ file text

Mỗi VU iteration chọn ngẫu nhiên một mã từ file `.txt` (coupon/card). File được load một lần lúc init bằng `open()`.

```javascript
// File: k6/src/utils.js (line 27-38)
export function getCouponCodes(filePath) {
    const couponCodes = open(filePath)
        .split(/\r?\n/)
        .map((v) => v.trim())
        .filter((v) => v.length > 0);
    if (couponCodes.length === 0) {
        throw new Error(
            `No coupon codes found in ${filePath}. Please provide a non-empty txt file.`,
        );
    }
    return couponCodes;
}
```

### 3.3 XML payload theo từng hệ thống CRM

| Script | Endpoint | XML shape |
|--------|----------|-----------|
| `k6-api-post-xml.js` | `TARGET_URL` (root) | `<Message><Coupon_ID>…</Coupon_ID><Partner_Code>…</Partner_Code></Message>` |
| `k6-rkcrm-cardinfo.js` | `TARGET_URL` | rkCRM `<Message Action="Get coupon info" Terminal_Type="…">` |
| `k6-rkcrm-cardholder.js` | `TARGET_URL` | rkCRM `<Message Action="Get card info" …><Card_Code>…</Card_Code><Include>…</Include>` |
| `k6-newapp-cardinfo.js` | `TARGET_URL/getcardinfoex` | `<ROOT><QRY Card="…" Restaurant="…" UnitNo="…"></QRY></ROOT>` |

Ví dụ rkCRM coupon info:

```javascript
// File: k6/k6-rkcrm-cardinfo.js (line 25-29)
function buildXmlPayload(couponCode) {
  return `<?xml version="1.0" encoding="utf-8" standalone="yes" ?>
<Message Action="${ACTION}" Terminal_Type="${TERMINAL_TYPE}">
<Coupon_ID>${couponCode}</Coupon_ID>
</Message>`;
}
```

### 3.4 Shell runner + env layering

`run.sh` đọc `k6/.env.k6`, merge với biến môi trường shell, optionally gọi `fetch-coupons.js`, rồi `k6 run`. Các script `run-rkcrm-*.sh` / `run-newapp-*.sh` set `TARGET_URL`, `COUPON_CODES_FILE`, `SKIP_COUPON_FETCH=1` rồi delegate sang `run.sh`.

### 3.5 Data prep từ SQL Server

`fetch-coupons.js` load `.env.k6`, query `CARD_COUPONS.COUPON_CODE`, ghi ra file text cho k6.

```javascript
// File: k6/fetch-coupons.js (line 42-47)
const connectionString = process.env.MSSQL_CONNECTION_STRING || "";
const query =
  process.env.COUPON_SQL_QUERY ||
  "SELECT COUPON_CODE FROM CARD_COUPONS WHERE COUPON_CODE IS NOT NULL AND LTRIM(RTRIM(COUPON_CODE)) <> '' AND DELETED = 0";
const outputFile =
  process.env.COUPON_CODES_FILE || path.join(__dirname, "coupon-codes.txt");
```

Mặc định `run.sh` set `SKIP_COUPON_FETCH=1` — dùng file có sẵn thay vì query DB mỗi lần chạy.

## 5. Key Flows

### Flow 1: Chạy load test rkCRM card info

```
Developer → run-rkcrm-cardinfo.sh
         → đọc TARGET_URL_RKCRM từ .env.k6
         → run.sh (SKIP_COUPON_FETCH=1, coupon-stg.txt)
         → k6 run k6-rkcrm-cardinfo.js
         → POST XML random Coupon_ID → rkCRM API
         → check status 2xx + thresholds
```

### Flow 2: Refresh coupon codes từ SQL Server

```
Developer → npm run fetch-coupons
         → fetch-coupons.js đọc MSSQL_CONNECTION_STRING + COUPON_SQL_QUERY
         → SELECT COUPON_CODE FROM CARD_COUPONS …
         → ghi COUPON_CODES_FILE (default coupon-codes.txt)
         → dùng file cho k6 scripts
```

### Flow 3: Chạy test với env tùy chỉnh

```
npm test / npm run test2 / test3
  → source k6/.env.k6
  → k6 run -e TESTNAME=… -e SV=rkCRM k6/k6-test.js
  → (hiện tại k6-test.js chỉ load codes và log — chưa implement HTTP)
```

## 11. Getting Started (Developer Onboarding)

1. Cài **k6** và **Node.js 18+**
2. `npm install` ở project root
3. Copy/sửa `k6/.env.k6` — điền `TARGET_URL*` và `MSSQL_CONNECTION_STRING` (không commit credentials)
4. Chuẩn bị data: dùng `cp1.txt` / `customer-stg.txt` có sẵn, hoặc `npm run fetch-coupons`
5. Chạy một scenario: `npm run test-rkcrm-cardinfo`
6. Xem k6 summary (RPS, latency percentiles, fail rate) trong terminal
7. (Tuỳ chọn) Import `k6/dashboard.json` vào Grafana khi có Prometheus metrics từ k6 cloud/output

**Tips:**

- `run.sh` mặc định **bỏ qua** DB fetch (`SKIP_COUPON_FETCH=1`) — bật bằng `SKIP_COUPON_FETCH=0` nếu cần refresh data
- Kiểm tra `TARGET_URL` trong `.env.k6` không có ký tự thừa (ví dụ dấu `'` cuối dòng)
- File `k6-test.js` đang WIP — dùng các script `k6-rkcrm-*` / `k6-newapp-*` cho test thực tế
- `docs/codebase/*.md` là placeholder — chưa có .NET source để scan
