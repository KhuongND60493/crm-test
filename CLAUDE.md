## Golden Rule

**Claude MUST NOT modify source code immediately after receiving a new feature request.**

Every feature or bug fix MUST go through this exact flow:

```
ASK → SPEC → PLAN → CODE → REVIEW
```

| Phase  | Slash Command     | Output                          |
|--------|-------------------|---------------------------------|
| ASK    | `/ask`            | Requirement Clarification doc   |
| SPEC   | `/spec`           | `.ai/specs/SPEC-[name].md`      |
| PLAN   | `/plan`           | `.ai/plans/PLAN-[name].md`      |
| CODE   | `/code`           | Implementation + summary        |
| REVIEW | `/review`         | AI Code Review report           |

---

## Approval Rules

Claude can only write or modify source code when the user says **exactly**:

```
APPROVED CODE
```

- `APPROVED` alone = SPEC is approved → move to PLAN phase only.
- `APPROVED CODE` = PLAN is approved → move to CODE phase.
- Any other phrasing does NOT count as approval.

---

## Scope Control

- Do NOT refactor unrelated code.
- Do NOT change public APIs unless the SPEC requires it and the user explicitly approved it.
- Do NOT modify database schema unless explicitly approved.
- Do NOT change business logic outside the approved scope.
- Do NOT introduce new dependencies unless approved.
- If a new risk or blocker is found during implementation, **STOP and ask** before continuing.

## CodeGraph Integration

> Nếu project có `.codegraph/` directory — **CodeGraph đã được index**.
> Ưu tiên dùng CodeGraph MCP tools **trước khi** đọc source file trực tiếp.

### Khi nào dùng tool nào

| Task | Dùng | Không dùng |
|------|------|-----------|
| Tìm class/interface/method theo tên | `codegraph_search` | grep, glob |
| Hiểu một feature hoạt động end-to-end | `codegraph_explore` | đọc nhiều files |
| Tìm tất cả nơi implement/dùng một interface | `codegraph_find_refs` | grep |
| Biết impact trước khi refactor method X | `codegraph_impact` | đọc callers thủ công |
| Đọc implementation sau khi đã biết file path | Read tool | — |

### Query patterns hiệu quả (C# / .NET 6)

```
codegraph_explore: "How does [CommandName] handler work end to end?"
codegraph_explore: "Trace the flow from [ControllerName] to database"
codegraph_explore: "How is [EntityName] persisted?"
codegraph_search:  "[InterfaceName]"  → xác định file path → Read
codegraph_impact:  "[MethodName]"     → review impact trước khi sửa
codegraph_find_refs: "[IRepositoryName]" → tìm tất cả implementations
```

### Vẫn đọc docs/ trực tiếp cho

- `docs/project_overview.md` — architecture decisions, conventions, DB provider
- `docs/codebase/*.md` — bulk reference khi cần context rộng (DI registry, DB schema)
- `docs/tasks/tasks_*.md` — task scope và checklist
- `.ai/specs/`, `.ai/plans/` — SPEC/PLAN contracts

### Khi CodeGraph chưa có (không có `.codegraph/`)

Fallback về `/scan-codebase` workflow như bình thường.
Recommend: chạy `codegraph init -i` trong project root để index.

## Dual-AI Workflow (Cursor + Claude Code)

This project uses **two AI agents with distinct roles**:

| Agent | Role | Strengths |
|-------|------|-----------|
| **Cursor AI** | Implementer — writes code fast, stays in editor | Speed, file editing, `/implement` rule |
| **Claude Code** | Architect + Reviewer — designs, reviews, governs | Reasoning, planning, quality gate |

### Workflow Modes

**Mode A — Claude designs, Cursor implements:**
```
Claude: /ask → /spec → /plan → /generate-cursor-task
                                        ↓
Cursor:                          /implement docs/tasks/tasks_[name].md
                                        ↓
Claude:                          /handoff docs/tasks/tasks_[name].md
```

**Mode B — Cursor implements ad-hoc, Claude reviews:**
```
Cursor: /task [description] → /implement
                                   ↓
Claude:                    /review-cursor docs/tasks/tasks_[name].md
```

**Mode C — Claude designs AND implements (no Cursor):**
```
Claude: /ask → /spec → /plan → /code → /review
```

### Shared Files (read by both AIs)

| File | Purpose | Updated by |
|------|---------|------------|
| `docs/project_overview.md` | Codebase map | Cursor (`/generate-overview`) |
| `CLAUDE.md` | Claude's behavior config | Claude (`/sync-overview`) |
| `docs/tasks/tasks_*.md` | Task checklists | Both |
| `.ai/specs/SPEC-*.md` | Requirement contracts | Claude |
| `.ai/plans/PLAN-*.md` | Design contracts | Claude |
| `.ai/reviews/REVIEW-*.md` | Quality gate results | Claude |

### Sync Rule

After Cursor runs `/generate-overview`, always run `/sync-overview` in Claude Code
to keep `CLAUDE.md` Project Info in sync with `docs/project_overview.md`.
